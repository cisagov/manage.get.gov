This table represents the members of each portfolio in the registrar. There are separate records for each member/portfolio combination.

Each member is assigned one of two access levels: admin or basic. Only admins can manage member permissions and organization metadata.

{% flag "user_portfolio_permission_invitations" %} Adding a new user portfolio permission sends an invitation email if the user doesn't exist, and you have the option to send an email if the user does exist. {% else %} If you add or remove someone to a portfolio by using this table, those actions won’t trigger notification emails. {% endflag %}
